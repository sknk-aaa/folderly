# Claude Code Prompts for Folderly

このドキュメントは、Claude Code で Folderly を実装する際の標準プロンプト集です。
実装は 3 セッションに分けて行います。

---

## セッション分割方針

| セッション | 対象 | 内容 | 自分の作業 |
|---|---|---|---|
| **1回目** | Step 1〜10 | Core 層 + Shell 層 全体 | テスト pass 確認のみ |
| **2回目** | Step 11〜16 | WPF GUI + ローカライズ + Store 判定 | 実機で動作確認 |
| **3回目** | Step 17〜18 | MSIX + 統合テスト | VS2022 でビルド確認 |

---

## 🚀 Prompt 1: セッション1（Step 1〜10）

```
Folderly という Windows 向けデスクトップアプリを実装します。
完全な仕様は ./SPEC.md にあります。まず SPEC.md を熟読してください。

【共通ルール・全セッション通じて守ること】
1. SPEC.md が唯一の正。記載がない機能は作らない、記載に反する実装はしない。
2. コミットは Step ごとに 1 つ、英語メッセージ、`feat:` `fix:` `test:` `chore:` のプレフィクス付き。
3. テストは xUnit で書く。Section 9 に最低件数の指定あり、すべて pass させる。
4. ライブラリの選定は Section 2 の固定リストに従う。
5. 仕様に明記されていない細部は、Section 8 の制約に反しない範囲で最もシンプルな解釈を採用し、
   判断内容を CLAUDE.md に記録する。
6. 詰まった場合・仕様と矛盾する場合のみ止まって確認を求める。それ以外は止まらない。

【今回の範囲: Step 1〜10（Core 層 + Shell 層）】
Step 1  プロジェクトスケルトン
Step 2  Core 層 - フォルダテンプレート定義
Step 3  Core 層 - 画像調整ロジック
Step 4  Core 層 - テンプレート合成
Step 5  Core 層 - 画像→.ico 変換
Step 6  Core 層 - desktop.ini 操作
Step 7  Shell 層 - キャッシュ更新
Step 8  Core 層 - 履歴管理
Step 9  Core 層 - 保護機能
Step 10 Core 層 - ユースケース

Step 1 から Step 10 まで一気に実装してください。

【着手前に必ず実施すること】
A) SPEC.md を読み、以下を箇条書きで確認する（コードはまだ書かない）:
   - v1.0 で実装する機能（F-01〜F-17）の総数
   - Out of Scope のうち特に注意すべきもの 3〜5 個
   - 技術スタック（Section 2）の主要 5 項目
   - 絶対禁止事項（Section 8.1）のうち特に意識すべきもの 3 つ
B) 不明点・確認したい点を質問する。

A と B が終わったら実装に入ってください。

【完了報告】
Step 10 完了後に以下を報告:
- 全実装ファイル一覧（パス）
- 全テスト結果（件数、pass/fail）
- CLAUDE.md に記録した仕様判断ログの内容
- 詰まった点・次セッションへの申し送り事項
```

## 🔁 Prompt 3: セッション2（Step 11〜16）

```
SPEC.md と CLAUDE.md を読んでから、Step 11〜16 を一気に実装してください。

【今回の範囲: Step 11〜16（WPF GUI + ローカライズ + Store 判定）】
Step 11 WPF GUI - フォルダプレビューコントロール
Step 12 WPF GUI - 画像選択画面
Step 13 WPF GUI - メイン管理画面
Step 14 WPF GUI - 設定画面
Step 15 ローカライゼーション
Step 16 Store ライセンス判定

【共通ルール】
- SPEC.md が唯一の正。セッション1と同じルールを継続。

コントロールの XAML と ViewModel のバインディング定義を完了報告に含めてください。

【完了報告】
Step 16 完了後に以下を報告:
- 全実装ファイル一覧（パス）
- 全テスト結果（件数、pass/fail）
- CLAUDE.md に追記した仕様判断ログ
- 私が実機確認すべき動作チェックリスト
- 次セッションへの申し送り事項
```

## 🧪 Prompt 4: セッション3（Step 17〜18）

```
SPEC.md と CLAUDE.md を読んでから、Step 17〜18 を実装してください。

【今回の範囲: Step 17〜18（MSIX + 統合テスト）】
Step 17 MSIX パッケージング
Step 18 統合テスト・調整

【Step 17 の進め方（必ず守ること）】
MSIX はビルドエラーが出やすいため、以下の順序で進める:
1. Package.appxmanifest の雛形を生成して私に見せる（ここで一度止まる）
2. 私が VS2022 で読み込んで検証
3. 修正指示があれば対応
4. 確認が取れたら Step 17 完了とする

右クリックメニュー宣言（desktop4:FileExplorerContextMenus）は
SPEC.md Section 10.1 のサンプルに忠実に従ってください。

【Step 18】
Step 17 の確認が取れたら、Step 18（統合テスト・調整）を実施。
docs/TESTING.md の手動テストチェックリストが完成していることを確認する。

【完了報告】
- 全実装ファイル一覧（パス）
- 全テスト結果（件数、pass/fail）
- docs/TESTING.md の内容
- VS2022 でビルドするための手順（README に記載されているか確認）
```

## 🛑 Prompt 5: 詰まり対応・仕様外検討時

Claude Code が独自判断で進めようとしたら使う:

```
それは SPEC.md の記載とどう関係しますか？以下のどれですか:

(a) SPEC.md に明記されており、その通り実装する
(b) SPEC.md に明記されていないが、合理的な解釈で SPEC.md に沿う
(c) SPEC.md と矛盾するため、私の判断が必要
(d) SPEC.md の Out of Scope に該当するため実装しない

(b) の場合は判断理由を CLAUDE.md に記録して進める。
(c) の場合は選択肢を提示して私の判断を待つ。
(d) の場合は実装せずに次へ進む。
```

## 📋 運用メモ

- **コミット粒度**: 1 Step = 1 コミット。細かく切らせない
- **セッション2後の実機確認**: Step 12〜13（画像選択画面、メイン画面）は VS2022 でビルドして目視確認が必須
- **セッション3の Step 17**: Package.appxmanifest の確認待ちで一度止まるのは正常
- **VS2022 が必要なステップ**: Step 11 以降の WPF GUI、Step 17 の MSIX
